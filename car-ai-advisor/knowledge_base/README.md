# 智能汽车导购助手 - 知识库说明

## 概述

本知识库为"智能汽车导购助手"AI Agent 系统的核心数据层，包含企业级汽车行业知识。知识库覆盖车型参数、用户评价、购车指南、保险保养、行业趋势、技术术语等维度，为 RAG（检索增强生成）管线提供高质量的结构化和非结构化数据。

## 知识库结构

```
knowledge_base/
├── README.md                  # 本文件
├── raw/                       # 原始数据（经过人工审核和标注）
│   ├── vehicles.json          # 车型结构化数据（16款主流车型）
│   ├── reviews.json           # 用户评价数据（10条详细评价）
│   ├── glossary.json          # 行业术语词典（20+核心术语）
│   ├── guides/                # 购车/用车知识指南
│   │   ├── purchase_guide.md  # 购车完全指南（6章，~4000字）
│   │   ├── insurance.md       # 保险完全指南（5章，~2500字）
│   │   ├── maintenance.md     # 保养完全指南（6章，~3000字）
│   │   └── ev_basics.md       # 新能源核心知识（5章，~3500字）
│   ├── industry/              # 行业报告与分析
│   │   ├── market_2026.md     # 2026市场格局分析（4章，~2500字）
│   │   └── tech_trends.md     # 技术趋势分析（5章，~3500字）
│   └── faq/                   # 常见问题
│       └── buying_faq.md      # 购车FAQ（14个高频问题，~3000字）
```

## 数据质量标准

### 结构化数据（JSON）
- ✅ 字段完整性：所有必填字段100%填写
- ✅ 类型一致性：数值/字符串类型统一
- ✅ 枚举值规范：价格区间、动力类型等使用统一标签
- ✅ 竞争关系标注：每款车型标注3个主要竞品

### 非结构化数据（Markdown）
- ✅ 信息密度：实用信息为主，无营销话术
- ✅ 时效性标注：每篇标注"最后更新"日期
- ✅ 引用来源：关键数据注明出处
- ✅ 分层结构：清晰的H2/H3标题层级，便于分块检索
- ✅ 表格丰富：复杂信息用表格呈现，提高可读性

## 与 RAG 管线的对接

### 数据分块策略
- **车型JSON**：每款车为一个独立chunk（直接按ID切分）
- **指南Markdown**：按H2标题切块（每块约500-800字）
- **FAQ**：每个Q&A为一个chunk
- **术语词典**：每个term为一个chunk

### 元数据标注建议
每个chunk应标注：
- `source_type`: vehicle / review / guide / industry / faq / glossary
- `brand`: 涉及品牌（车型类）
- `category`: 内容分类（三电/智驾/价格/安全/保养等）
- `price_range`: 涉及价格区间（用于价格过滤）

### Embedding 模型推荐
- 中文：BAAI/bge-base-zh-v1.5（1024维，本地部署）
- 英文/多语言：BAAI/bge-m3（1024维，多语言）
